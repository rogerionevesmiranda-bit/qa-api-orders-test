Bug: API aceita payload inválido ao criar pedido

 Ambiente
- API de pedidos (ambiente simulado)
- Método: POST /orders

 Descrição
Ao enviar um payload inválido para a criação de pedidos, a API aceita a requisição e retorna sucesso, quando o comportamento esperado seria rejeitar o pedido com erro de validação.

Passos para reproduzir
1. Enviar uma requisição POST para o endpoint `/orders`
2. Utilizar o seguinte payload inválido:
   ```json
   {
     "customer": "João Silva",
     "total": -50

   }
Enviar a requisição

Resultado esperado >

A API deve retornar erro de validação

Status HTTP 400 (Bad Request)

Mensagem clara indicando campos inválidos

Resultado atual >

A API retorna status de sucesso

O pedido é aceito mesmo com payload inválido

Evidências >

Teste automatizado de payload inválido configurado no Postman

Script de validação no post-response esperando status 400

Severidade >

Alta - Esse problema permite a criação de pedidos com dados inconsistentes, o que pode gerar impactos financeiros, erros de processamento e problemas de integridade de dados.
